# Inputs / Variables

You can customize this module using the following variables defined in variables.tf:
## 📌 General & Compute

`group_name` (Required): The given name for this group of instances.

`default_ami_value` (Required): Pass any AMI ID if you want to customize at the AMI level.

`instance_count`: The total number of EC2 instances used to create this load balancer (Default: 0).

`instance_type`: The default EC2 instance type if no argument is passed (Default: "t3.micro").

`default_region`: The AWS region for the resources (Default: "us-east-1") - redundacy with one declared in the `providers.tf`, will be deleted in later update.

## 🌐 Networking

`vpc_id`: ID of the VPC for this resource; otherwise, it falls to the default VPC (Default: "").

`machine_net`: The subnet ID for the main EC2 machine network (Default: "").

## 🛡️ Bastion Host Setup

`bastion_host`: Indicates whether to provision a bastion host (1 for yes, 0 for no) (Default: 0).

`bastion_net`: The subnet ID for the bastion host (Default: "").

## 🔑 Security & SSH Access

`generated_new_ssh_key`: Optional, choose to generate a new SSH key automatically on run (1 for yes, 0 for no) (Default: 1).

`pre_created_ssh_key`: Specify whether to use a pre-created SSH key (1 for yes, 0 for no) (Default: 0).

`open_publicy_ssh`: The option to open SSH to the public (1 for yes, 0 for no) (Default: 0).

`key_alogrithm`: The algorithm used to generate the SSH key pair. Must be RSA, ECDSA, or ED25519 (Default: "RSA").

`key_bits`: The number of bits used in the key generation process (Default: 4096).
